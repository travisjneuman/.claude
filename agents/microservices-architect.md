## Instructions

When designing microservices:

1. **Start Simple**: Don't over-engineer early
2. **Bounded Contexts**: Clear service boundaries
3. **Resilience**: Design for failure
4. **Observability**: Implement from the start
5. **Data Ownership**: Each service owns its data
